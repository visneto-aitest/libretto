---
title: One-shot workflow generation
---

> Give your agent a goal and a URL, and get back a complete browser automation script.

One-shot generation is the fastest path from idea to working automation. You describe what you want in a single prompt, and your coding agent — armed with the Libretto skill — opens a browser, explores the site, and writes a complete workflow file. No recording, no hand-holding.

### Steps

<Steps>
  <Step title="Prompt your agent">
    Describe the **site**, the **data or action** you want, and the **shape of the output**. The more specific you are, the better your results.

    > Use the Libretto skill. Go on Hacker News and scrape the first 10 posts for title, URL, score, author, and number of comments.

    Include "Use the Libretto skill" in your prompt so your coding agent uses Libretto's CLI tools instead of writing a Playwright script from scratch.

  </Step>

  <Step title="Log in when the browser opens (if needed)">
    The agent opens a browser window on your screen. If the site requires authentication, the agent will ask you to log in manually. Do that, then tell the agent you're done.

    ```bash  theme={null}
    # What the agent runs behind the scenes
    npx libretto open https://news.ycombinator.com --headed --session one-shot-flow
    ```

    <Note>
      If the site needs login, make sure your prompt doesn't say `--headless` — the agent needs to open a visible browser so you can authenticate. You can mention "I need to log in first" in your prompt to be explicit.
    </Note>

  </Step>

  <Step title="The agent explores the page">
    The agent takes snapshots of the page and tests selectors against the live DOM. You'll see it running commands like `npx libretto snapshot` and `npx libretto exec` in your terminal. The browser window stays visible so you can watch what's happening.

    You don't need to do anything during this phase, but if the agent seems stuck or heading in the wrong direction, you can interrupt and redirect:

    > That's the wrong section — the posts are in the main feed, not the sidebar.

  </Step>

  <Step title="Review the generated workflow file">
    The agent writes a TypeScript file that exports a `workflow()`. Here's roughly what to expect:

    ```typescript  theme={null}
    import { workflow } from "libretto";

    type Output = {
      posts: Array<{
        title: string;
        url: string;
        score: number;
        author: string;
        commentCount: number;
      }>;
    };

    export const hnTopPosts = workflow<{}, Output>(
      "hnTopPosts",
      async (ctx): Promise<Output> => {
        const { page } = ctx;

        await page.goto("https://news.ycombinator.com");
        await page.waitForSelector(".athing");

        const rows = await page.locator(".athing").all();
        const posts = [];

        for (const row of rows.slice(0, 10)) {
          const title = await row.locator(".titleline > a").textContent();
          const url = await row.locator(".titleline > a").getAttribute("href");
          // ... extract score, author, comment count from subtext row
        }

        return { posts };
      },
    );
    ```

    Things to check:
    - **Output type** matches what you asked for
    - **Selectors** look reasonable (class names, data attributes, or ARIA roles — not brittle positional selectors)
    - **Pagination** logic is present if you asked for more items than fit on one page

  </Step>

  <Step title="Check the validation run">
    The agent runs the workflow headless and shows you the output. Check that the returned data looks right — correct field names, real values, expected number of items.

    ```bash  theme={null}
    # What the agent runs
    npx libretto run ./hn-top-posts.ts hnTopPosts --headless
    ```

    If the output is wrong or incomplete, tell the agent what's off:

    > The reactions count is always 0 — it looks like it's reading the wrong element.

    The agent will re-inspect the page and fix the issue.

  </Step>
</Steps>

### When to use this approach

- You're automating a site for the first time and don't know its structure
- The task is self-contained and clearly scoped ("get the first 10 posts")
- You want a working script with minimal back-and-forth
- You don't need to demonstrate the workflow manually first

### Tips

<Tip>
  After you log in, tell the agent to save the session with `npx libretto save`.
  Future runs can restore your login state so you don't have to authenticate
  every time.
</Tip>

<Note>
  Make sure your coding agent has the Libretto skill installed before starting.
  Run `npx libretto setup` in your project root to set up the workspace and
  configure snapshot analysis if you haven't already.
</Note>
