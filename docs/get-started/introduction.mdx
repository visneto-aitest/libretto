> Libretto is the AI toolkit for building and maintaining browser automations.

At [Saffron Health](https://saffron.health), we integrate with dozens of healthcare EHRs and payor portals that don't have APIs. That means browser automation: logging into legacy web apps, navigating complex UIs, and extracting data from pages that were never designed for programmatic access. We built Libretto to make that work reliable and maintainable.

Libretto gives your coding agent a live browser and a token-efficient CLI to inspect pages, capture network traffic, record user actions, and debug workflows, all without flooding the agent's context window with raw HTML. It's open-source so other teams dealing with similar problems can use it too.

## Get started

Run this in your project directory:

```bash theme={null}
npm init libretto@latest
```

This walks you through installing Libretto, choosing an AI provider, setting up the `.libretto/` directory, and downloading Chromium if needed. It handles everything for you.

<Note>
  Run this yourself in a terminal, not through an agent. It may prompt for
  input.
</Note>

If you want to change your AI provider or other settings later, see the [Configuration](./configuration) page.

Once setup is complete, load up your favorite coding agent and try a prompt:

```text Example prompt
Use the Libretto skill. Go to Craigslist and scrape the first 10 listings in the "free stuff" section for title, location, and link.
```

## Core capabilities

- **Inspect live pages:** Snapshot any page and get a token-efficient summary of selectors, interactive elements, and visible state.
- **Capture network traffic:** Record every HTTP request/response to a structured JSONL log for reverse-engineering site APIs.
- **Record user actions:** Capture clicks, form fills, and navigation in the browser so your agent can turn them into Playwright scripts.
- **Debug broken workflows:** Keep the browser open on failure so your agent can inspect, patch, and re-run without restarting.

## Common commands

```bash
npx libretto open https://example.com --session my-session
npx libretto snapshot --session my-session --objective "Find the submit button" --context "I just opened the homepage."
npx libretto exec --session my-session "return await page.url()"
npx libretto run ./integration.ts main
```
